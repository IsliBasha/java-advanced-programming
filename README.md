# Java Advanced Programming — Assignment Portfolio

A collection of **nine independent Maven projects** built for the Java Advanced
Programming course. Each project lives in its own directory, builds on its own,
and ships with a dedicated `README.md` describing its design and a short
*Notes for the Professor* section.

The portfolio is split into two groups:

- **Console applications (01–05)** — pure Java demonstrating the Collections
  Framework, `Optional`, and the Streams API. Totals and averages are computed
  with explicit loops (no `sum()`/`average()` aggregate shortcuts), as required.
- **REST APIs (06–09)** — Spring Boot 3 services with a layered architecture
  (controller → service → repository → entity), Bean Validation, centralized
  exception handling, and JPA persistence.

All nine projects target **Java 21**.

---

## Skills demonstrated

**Language & core Java** — Java 21, `record` types, enhanced `switch` and pattern
matching, `Optional`, the Collections Framework (`List`/`Set`/`Map`), and the
Streams API.

**Backend & frameworks** — Spring Boot 3.3, Spring Web (REST), Spring Data JPA /
Hibernate, Bean Validation, layered architecture (controller → service →
repository → entity), DTO mapping, and centralized `@RestControllerAdvice`
exception handling.

**Persistence** — MySQL at runtime with JPA schema generation and `data.sql`
seeding; in-memory H2 for tests.

**Testing & tooling** — JUnit 5 (89 tests across the nine projects), Maven, Git.

## Build & test

Every project is self-contained. To build and test one project:

```bash
cd Assignment01-OnlineCourseAnalytics
mvn clean test
```

To build and test **all nine** from this directory:

```bash
for d in Assignment*/; do
  echo "=== $d ==="
  ( cd "$d" && mvn -q clean test ) || { echo "FAILED: $d"; break; }
done
```

## Run

### Console applications (01–05)

Each console app has a configured main class and runs via the Exec plugin:

```bash
cd Assignment01-OnlineCourseAnalytics
mvn -q clean compile exec:java
```

The `main()` method seeds demonstration data and prints formatted output to the
console.

### REST APIs (06–09)

1. Set your MySQL credentials in `src/main/resources/application.properties`
   (the committed file uses a placeholder password).
2. Start the service:

```bash
cd Assignment06-CourseEnrollmentAPI
mvn spring-boot:run
```

The schema is auto-generated by JPA and seeded from `data.sql` on startup.
Each API listens on its own port (see the table above), so several can run
simultaneously.

## Conventions shared across all projects

- **Java 21** with modern language features (records, enhanced `switch`,
  pattern matching, text blocks where they improve clarity).
- **Immutable models** — console apps use `record` types with `withX()` copy
  methods; APIs separate request/response DTOs from JPA entities.
- **Layered architecture** for APIs — controllers stay thin; all business rules
  live in `@Service` classes; data access is through Spring Data repositories.
- **Centralized error handling** — each API has a `@RestControllerAdvice` that
  returns structured JSON for validation failures and domain exceptions.
- **Javadoc** on public types, **JUnit 5** test suites, and a per-project
  `.gitignore`.
